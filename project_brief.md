PulseCoach
A Personal Android Training Companion for the Polar H10
Project Brief & AI Collaboration Guide (CLAUDE.md)

Part 1 — Project Brief
1.1  Executive Summary
PulseCoach is a personal Android application that connects to a Polar H10 heart rate monitor via Bluetooth LE and delivers a training experience built around three core ideas: real-time feedback through customizable HR zones, session-level effort analysis through caloric output tracking, and predictive intelligence that estimates end-of-session caloric output based on your personal historical patterns.
This project is simultaneously a practical fitness tool and a portfolio piece demonstrating end-to-end software development — from hardware integration and mobile UI through time-series analytics and lightweight machine learning.

1.2  Problem Statement
Commercial heart rate apps like Polar Flow are designed for mass-market appeal. They are opinionated, subscription-gated, and offer little flexibility. Specifically:
•	HR zone thresholds are fixed or require account setup — no in-session tuning.
•	Caloric output is buried or absent from the live session view.
•	No app offers a personalized projection of where your session is heading based on your own data.
•	Export options are locked behind ecosystems (Garmin Connect, Polar Flow) rather than portable formats like CSV.
PulseCoach solves all four pain points for a single user: you.

1.3  Goals & Non-Goals
Goals
•	Connect reliably to the Polar H10 via BLE and stream real-time HR.
•	Display HR on a live scrolling graph with color-coded zone bands.
•	Allow the user to define custom HR zone thresholds (stored in app settings).
•	Calculate and display live calories/minute using the Keytel formula.
•	Record full sessions to local storage and export as CSV.
•	Project estimated end-of-session caloric output given a user-defined session duration.
•	Improve projections over time using personal session history.
Non-Goals
•	No cloud sync, no accounts, no subscriptions.
•	Not designed for multiple users or device types beyond the H10.
•	No GPS, cadence, or power meter integration in v1.

1.5  Tech Stack
•	Kotlin Language:
◦	Modern, concise, Android-native. Coroutines handle async BLE streams cleanly.
•	Jetpack Compose UI:
◦	Declarative UI — easier to learn than XML layouts, aligns with current Android best practices.
•	Polar BLE SDK (official, open source, Gradle dependency) BLE:
◦	Handles Bluetooth pairing, reconnection, and the Polar-proprietary data protocol.
•	Vico (Compose-native chart library) Charting:
◦	Smooth live data updates, zone coloring, composable API.
•	Room (SQLite ORM) for session history Persistence:
◦	Structured session storage. CSV export via Kotlin file I/O.
•	Keytel et al. (2005) — HR-based estimation Calorie Formula:
◦	Inputs: HR, age, weight, biological sex. Solid accuracy for sustained cardio effort.
•	In-session curve fitting + historical session weighting Projection (Phase 3):
◦	Start: polynomial extrapolation of current session trajectory.
◦	After 10+ sessions: weighted blend of current trajectory and personal historical patterns.

1.7  Analytics & Portfolio Angle
This project tells a story that is rare in business analytics portfolios: hardware-to-insight in a self-contained system. The analytical layers stack naturally:
•	Descriptive: Session summaries, HR distribution by zone, cal/min over time.
•	Diagnostic: Why did this session burn more than last Tuesday? (HR zone distribution shift)
•	Predictive: Given my current trajectory and past sessions, where will I land by minute 60?
•	Prescriptive: What pace should I hold to hit a caloric target? (Phase 3+)
Each phase adds a new analytical tier. The dataset is entirely yours — generated by your own body — which is a compelling differentiator in a portfolio context.
